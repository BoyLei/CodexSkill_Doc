# Codex 进程清理规则

Codex 只能关自己明确为当前操作一次性任务启动的进程，且该操作已执行完毕。

## 规则

### 主动关
- 临时测试/分析脚本 → 验证完成即关
- 当前回合启动的 `exec_command` 长时间会话 → 回合结束前确保已退出或关闭

### 不关
- 一次性命令（跑完自动退）
- 后台服务（dev server、文件监听等）→ 除非用户明确说"关掉"
- 前几轮或其他会话遗留的进程
- 插件 MCP 进程（由 Codex 桌面端管理）

## 边界
- 不主动替用户做判断，不确定使用状态的进程一律不碰
- 如果启动了用户可能继续用的服务，只告知 PID，不代替关闭

## 清理脚本
`scripts/cleanup-codex.ps1` 实现了上述规则的具体扫描和清理逻辑。

用法: `.\scripts\cleanup-codex.ps1 [-Force] [-ListOnly]`
- 不加参数：交互式，列出可清理项，确认后执行
- `-Force`：跳过确认，直接清理已知安全项
- `-ListOnly`：只列出进程状态，不执行清理
